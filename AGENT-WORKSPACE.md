# multi-gcal-mcp Workspace

Multi-Google-Calendar aggregation platform with agentic MCP access, web calendar view, and a future CalDAV sync layer.

## Product Areas

Four distinct products in this monorepo:

1. **`@multi-gcal/core`** — Shared data layer: OAuth token storage, Google Calendar API access, per-account sub-calendar filters
2. **`@multi-gcal/mcp`** — MCP server (stdio + StreamableHTTP) exposing calendar tools to Claude
3. **`@multi-gcal/web`** — Dark-mode weekly calendar web UI (read-only aggregated view)
4. _(future)_ **`@multi-gcal/caldav`** — CalDAV server transparently merging N accounts into one feed

### Known Account Issues
- `seth@react.foundation` and `seth@expo.dev` fail OAuth refresh with `unauthorized_client`
- Root cause: Google Workspace admin restrictions on third-party OAuth apps (not a code bug)
- Fix requires Workspace admin approval of the OAuth client ID

### Sub-Calendar Filters
- Per-account calendar ID allowlist stored in `calendarFilters` key in tokens.json
- Default (unset) → `['primary']`
- MCP tools: `gcal_set_calendar_filter`, `gcal_get_calendar_filters`, `gcal_list_calendars`

## Architecture Decisions

### Decided
- Bun monorepo with `workspace:*` deps (ADR pending)
- JS (not TS) currently — TypeScript migration needed
- StreamableHTTP transport for Claude Desktop "custom connector" UI
- Cloudflare named tunnel (not ngrok/localtunnel) for stable public HTTPS

### Open / Pending
- TypeScript migration across all packages
- CalDAV server design (iCal feed vs full CalDAV protocol)
- Cross-account conflict detection / busy-time merging
- Authentication for web UI (currently unauthenticated on :3747)
